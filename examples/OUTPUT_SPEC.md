# auto-dao 学习输出文件规范与实现顺序

> 本文档基于对 6 个实际案例（`examples/learning-history/`）和项目指导文件（`CLAUDE.md`、`SKILL.md`、模板文件）的逐文件分析归纳而成。所有规则均有实例依据，无臆造内容。

---

## 一、目录结构规范

### 1.1 会话目录命名

**格式**：`{topic_name}_{timestamp}`

| 实际案例 | topic_name | timestamp 格式 |
|---------|-----------|---------------|
| `被拒绝的勇气_2026-04-06` | 中文主题名 | `YYYY-MM-DD` |
| `adlerian-courage_2026-04-06-10-06` | 英文短横线命名 | `YYYY-MM-DD-HH-MM` |
| `一元函数微分学_2026-04-06-21-30` | 中文主题名 | `YYYY-MM-DD-HH-MM` |
| `attention-transformer_2026-04-06-10-52` | 英文短横线命名 | `YYYY-MM-DD-HH-MM` |
| `videocut-skills_2026-04-06` | 英文短横线命名 | `YYYY-MM-DD` |

**规范化规则（v2.0）**：

| 组成 | 规则 | 示例 |
|------|------|------|
| `topic_slug` | 全小写 ASCII，单词以连字符分隔；中文主题使用英文翻译或拼音 | `attention-transformer`、`calculus-derivatives` |
| `timestamp` | 固定 `YYYY-MM-DD-HH-MM` 格式 | `2026-04-06-10-06` |

> **历史兼容说明**：现有示例中存在中文 topic_name（如 `被拒绝的勇气`）和简化时间戳（如 `YYYY-MM-DD`），这些为早期案例，保留不做重命名。新会话必须遵循上述规则。

**早期案例的观察记录**：
- `topic_name` 从学习资料中自动提炼，可中文可英文
- `timestamp` 在 SKILL.md 中规定为 `YYYY-MM-DD-HH-MM`，但实际案例中存在简化为 `YYYY-MM-DD` 的情况

### 1.2 目录树

```
learning-history/{topic_name}_{timestamp}/
├── summary.md            # 学习摘要（必有）
├── roadmap_status.md     # 学习路线图状态（必有）
├── report.md             # 学习报告（仅学习完成后生成）
└── lessons/
    ├── lesson_1.md       # 第 1 课
    ├── lesson_2.md       # 第 2 课
    └── ...               # 按需递增
```

**案例验证**：
- 6 个案例均包含 `summary.md`、`roadmap_status.md`、`lessons/` 三个要素
- 无案例包含 `report.md`（因均未完成全部课程）
- `lessons/` 下文件数量：5 个案例各含 1 个 lesson 文件，`一元函数微分学_2026-04-06-21-30` 含 2 个

## 二、三大核心文件规范

### 2.1 summary.md（学习摘要）

**作用**：追踪学习的核心结论、掌握状态、难点和进度，是"全局记忆"。

**固定结构**（基于模板 `summary-template.md` 和 6 个案例归纳）：

```markdown
# 学习摘要 (Learning Summary)

**主题**: {topic_name}
**开始时间**: {timestamp}
**最后更新**: {timestamp}

---

## 核心结论                    ← 学习过程中逐步填充，非预写
## 已掌握知识点
### ✅ 完全掌握               ← 通过 ≥80% 习题的知识点
### 🔄 基本掌握               ← 部分正确的知识点
## 待攻克难点
### 🎯 当前重点               ← 含 **状态** 和 **建议** 字段
### ⏳ 待解决                 ← 偏题问题暂存区（HTML 注释说明用途）
## 学习进度                   ← 已完成课时 / 总课时、认知层级、进度百分比
## 学习路径                   ← 用 ASCII 箭头图示，标注 [已完成]/[当前]/[待学习]
## 关键笔记
### 重要公式/定理（或"重要概念"） ← 理科用"公式/定理"，人文用"概念"
### 常见误区
### 实践技巧
## 下次学习计划               ← 1-3 条具体行动
```

**状态演变规则**（来源：`一元函数微分学_2026-04-06-21-30` 的 summary.md 对比其他初始案例）：

| 阶段 | 核心结论 | 已掌握知识点 | 学习进度 |
|------|---------|-------------|---------|
| 初始化（0 课完成） | "学习过程中逐步填充" | "暂无"或"待填写" | `0 / N`，`0%` |
| 学习中（1+ 课完成） | 填入具体结论文字 | 列出具体知识点和公式 | `K / N`，`K/N × 100%` |

### 2.2 roadmap_status.md（学习路线图）

**作用**：精细追踪每个知识点的状态、里程碑、行动计划，是"项目管理仪表盘"。

**固定结构**：

```markdown
# 学习路线图状态 (Roadmap Status)

**主题**: {topic_name}
**时间戳**: {timestamp}

---

## 整体进度                   ← 含进度条 ASCII 可视化
## 认知坐标                   ← 表格：当前课时、认知层级、掌握程度、难度感知
## 知识点追踪                  ← 按模块分组，每个知识点一行，含状态和测试结果
## 里程碑                     ← Checkbox 列表，含最终目标（🎁 标记）
## 学习历史                   ← 表格：日期、Lesson、状态、备注
## 下一步行动                  ← 即时任务 + 本周目标
## 阻塞与风险                  ← 表格：风险、影响、解决方案
## 更新日志                   ← 按日期分组的变更记录
```

**知识点状态标记**（3 种，全部案例统一）：

| 标记 | 含义 | 出现条件 |
|-----|------|---------|
| ✅ 已掌握 | 通过该知识点的习题 | 习题正确率 ≥ 80% |
| 🔄 学习中 | 当前正在学习 | 正在进行的 Lesson 所涉及 |
| ⏳ 待学习 | 尚未涉及 | 未来 Lesson 的知识点 |

**进度条样式**（案例观察到两种）：

```
# 样式 1（空格填充）——被拒绝的勇气
[                                                  ] 0%

# 样式 2（Unicode 方块）——adlerian-courage、一元函数微分学
[░░░░░░░░░░░░░░░░░░░░] 0/5 Lessons
[████░░░░░░░░░░░░░░░] 20%
```

**模块划分规律**（6 个案例归纳）：

| 学习类型 | 模块数量 | 每模块知识点数 | 划分依据 |
|---------|---------|-------------|---------|
| 人文社科（被拒绝的勇气） | 4 | 2-4 | 书的主题章节 |
| 理科教材（一元函数微分学） | 3 | 4-5 | 数学概念递进关系 |
| 工科论文（attention-transformer） | 5 | 3 | 论文章节结构 |
| 开源项目（videocut-skills） | 6 | 2-3 | 项目模块/Skill 拆分 |

### 2.3 lessons/lesson_N.md（课程文件）

**作用**：单节课的完整内容——讲解 + 习题 + 反馈区，是"文件驱动学习"的核心载体。

**固定结构**（基于模板 `lesson-template.md` 和 7 个实际 lesson 文件归纳）：

```markdown
# Lesson {N}: {lesson_title}

**时间**: {timestamp}
**主题**: {topic_name}
**目标认知层级**: {bloom_level}

---

## 学习目标                    ← 2-4 条，以"你应该能够"开头
## 讲解内容                    ← 分 3-6 个子章节，含原文引用
## 课后习题                    ← 2-4 道题，每题后预留"**我的答案**："
## Reflection                 ← 认知难度 1-5 量表 + 需澄清内容
## 本节要点总结                 ← 3-6 条要点
## 下一课预告                  ← 简述下一课主题
```

## 三、讲解内容生成规范

### 3.1 引用格式（防幻觉核心机制）

**从案例中观察到的 3 种引用格式**：

#### 格式 1：资料原文块引用（高频，所有 lesson 均使用）

```markdown
> **资料原文**：
> "{从学习资料中直接摘录的原文}"
```

- **用途**：直接引用学习材料中的关键段落
- **验证**：7 个 lesson 文件中全部使用此格式，最少的有 4 处（videocut-skills），最多的有 10+ 处（一元函数微分学）

#### 格式 2：来源标注（高频，SKILL.md 强制要求）

```markdown
[来源：{章节或段落标题}]
```

- **用途**：标注讲解内容的出处
- **实例**：
  - `[来源：第一章"现在，让我们来谈谈幸福"]`（被拒绝的勇气）
  - `[来源：5.1.1 导数的概念]`（一元函数微分学）
  - `[来源：1 Introduction]`（attention-transformer）
  - `[来源：\`README.md\` §为什么做这个？]`（videocut-skills）

#### 格式 3：资料未覆盖标注（SKILL.md 要求，案例中未实际触发）

```markdown
[⚠️ 当前资料未涉及此内容]
```

### 3.2 讲解风格适配（基于用户背景）

**从 6 个案例中归纳的适配策略**：

| 适配维度 | 案例依据 | 具体表现 |
|---------|---------|---------|
| **年级适配** | 一元函数微分学 lesson_1 出现"教学提示（针对K12教学）" | 因 background.md 提到"家教"相关背景 |
| **专业适配** | 被拒绝的勇气 lesson_1 习题 4 "教学思考（针对家教背景）" | 习题与用户职业背景结合 |
| **深度适配** | attention-transformer 使用论文原文英文 + 中文讲解 | 适应工科论文阅读需求 |
| **类比适配** | 一元函数微分学用"平均速度"类比差商 | 用已知概念引入新概念 |

### 3.3 反事实论证模式

**触发条件**：用户在 Step 4 中选择"开启反事实论证"。

**从案例中观察到的两种实现方式**：

**方式 1：内联式**（adlerian-courage lesson_1）
- 在每个核心概念讲解后直接追加"**反事实论证**："段落
- 格式：先提出反事实假设，再推导逻辑矛盾
- 该案例有 3 处反事实论证，均出现在讲解子章节内

**方式 2：独立章节式**（attention-transformer lesson_1、一元函数微分学英文版）
- 设置独立的 `### ⚡ 反事实论证（Counterfactual Reasoning）` 章节
- 集中讨论核心结论的反面假设
- 包含"边缘案例"分析

**未开启反事实论证的案例**（被拒绝的勇气、一元函数微分学中文版、videocut-skills）：
- 不包含任何"反事实"相关段落

### 3.4 自检摘要

SKILL.md 要求 Lesson 生成后执行来源自检，并在通知中附加摘要。

**案例验证**：`adlerian-courage` lesson_1 末尾有：
```
🔍 自检完成：12 处已标注来源，0 处标注"当前资料未涉及"
```
其他案例的 lesson 文件中未在文件内包含此行（符合 SKILL.md 的"仅在通知中附加"要求）。

## 四、课后习题规范

### 4.1 习题数量与类型

| 案例 | 习题数 | 题型分布 |
|-----|-------|---------|
| 被拒绝的勇气 lesson_1 | 4 道 | 概念辨析、理解应用、反思分析、教学思考 |
| adlerian-courage lesson_1 | 4 道 | 概念辨析、深度理解、情境应用、反事实思考（选做） |
| 一元函数微分学 lesson_1 | 4 道 | 理解本质、几何应用、判断可导性、关系辨析 |
| 一元函数微分学 lesson_2 | 4 道 | 概念理解、公式应用、链式法则、导数与微分 |
| 一元函数微分学（英文）lesson_1 | 4 道 | 定义理解、单侧导数、定义法求导、关系判断 |
| attention-transformer lesson_1 | 3 道 | 用自己的话解释、概念辨析、推理题 |
| videocut-skills lesson_1 | 3 道 | 用自己的话解释、数据流追踪、角色判断 |

**归纳**：
- 习题数量：2-4 道（符合 SKILL.md 的"2-4 道"规定）
- 人文类偏重反思和应用，理科类偏重计算和证明，工科/项目类偏重理解和追踪

### 4.2 习题固定格式

```markdown
### 习题 {N}：{题目类型描述}

{题目描述}

[来源：{章节标题}]

**我的答案**：

---
```

**关键约束**：
- `**我的答案**：` 是固定格式关键词，不得更改（SKILL.md 明确规定）
- 每道习题必须标注 `[来源]`
- 习题后必须有 `---` 分隔线
- 开启反事实模式时，可追加"选做"标记的反事实思考题（见 adlerian-courage）

### 4.3 已填写答案的样式

**来源**：`一元函数微分学_2026-04-06-21-30` lesson_1 是唯一包含已填写答案的案例。

答案格式特征：
- 直接在 `**我的答案**：` 后空一行开始书写
- 可包含 LaTeX 公式（`$...$` 和 `$$...$$`）
- 可包含引用教材原文的具体行号（如"教材定义（第8218行）"）
- 可包含多级结构（标题、列表、公式块）
- 故意错误的答案用括号注明"（此处是让 AI 故意写错...）"

## 五、跨语言教学规范

### 5.1 案例对比

| 案例 | 材料语言 | 教学语言 | 跨语言？ |
|-----|---------|---------|---------|
| 被拒绝的勇气 | 中文 | 中文 | 否 |
| adlerian-courage | 中文 | 中文 | 否 |
| 一元函数微分学_21-30 | 中文 | 中文 | 否 |
| **一元函数微分学_22-00** | **中文** | **英文** | **是** |
| attention-transformer | 英文 | 中文 | 是（材料英文，讲解中文） |
| videocut-skills | 中文 | 中文 | 否 |

### 5.2 英文教学输出特征（一元函数微分学_22-00）

- **标题**：`# Lesson 1: The Concept and Geometric Meaning of Derivatives`
- **元数据**：`**Time**:`, `**Topic**:`, `**Target Cognitive Level**:`
- **章节标题**：`## Learning Objectives`, `## Content`, `## Practice Problems`
- **习题提示**：`> Please fill in your answers in the "My Answer" section...`
- **答案关键词**：`**My Answer**:` （注意：模板规定结构关键词保持固定，但此案例使用了英文）
- **来源标注**：`[Source: Section 5.1.1]`、`[Source: Definition 5.1]`
- **原文引用**：`> **Original Text**:`（替代了 `> **资料原文**：`）
- **Reflection**：`**Cognitive Difficulty** (1-5):`, `**Which parts need clarification?**:`
- **要点总结**：`## Key Takeaways`

### 5.3 中文教学（英文材料）特征（attention-transformer）

- 章节标题保持中文：`## 学习目标`、`## 讲解内容`、`## 课后习题`
- 引用原文保持英文：`> **资料原文**：\n> "Recurrent neural networks..."`
- 讲解正文使用中文
- 来源标注使用英文章节名：`[来源：1 Introduction]`

## 七、实现顺序（Pipeline）

基于 SKILL.md 的 5 步流水线和案例验证：

### Step 1: 文件格式预处理

```
输入 → 判断类型 → [GitHub 链接?] → git clone + 生成 overview
                  [目录?] → 复制 + 生成 overview  
                  [可读文件?] → 直接使用
                  [需转换?] → 调用 everything-to-markdown
                  [不可读?] → 报错终止
       → Markdown 质量检测 → [有问题?] → 调用 markdown-refiner
       → 记录 processed_path
```

### Step 2: 初始化与历史检索

```
提炼 topic_name → 记录 timestamp → 扫描 learning-history/ 
→ 读取匹配目录的 summary.md → 构建选项列表
→ ⛔ BLOCKING: 等待用户选择"继续"或"全新学习"
```

### Step 3: 上下文恢复

```
[全新学习] → 创建目录 → 初始化 summary.md（从模板） 
                       → 初始化 roadmap_status.md（从模板）
                       → 进入 Step 4

[继续历史] → 读取 summary.md + roadmap_status.md 
           → 检查当前 lesson 文件状态
           → [不存在] → 生成新 lesson
           → [存在但未答] → ⛔ BLOCKING: 通知用户去填写
           → [存在已答] → 执行答案检查
```

### Step 4: 文件驱动交互式学习（循环）

```
⛔ BLOCKING: 询问是否开启反事实论证（仅首次）
→ 4.1 逆向设计：读取 bloom-taxonomy.md + socratic-method.md + background.md
                 → 确定目标布卢姆层级
→ 4.2 生成 lesson_{N}.md：
    - 从 lesson-template.md 出发
    - 填入讲解内容（含 > **资料原文**： 引用）
    - 填入 2-4 道习题（含 **我的答案**： 预留区）
    - 执行自检：标注 [来源] 和 [⚠️ 当前资料未涉及]
→ 通知用户文件路径
→ ⛔ BLOCKING: 等待用户说"已完成"

→ 4.3 答案检查：
    - 读取 lesson 文件，提取"我的答案"区域
    - ≥80% 正确 → 进入下一课
    - <80% → 苏格拉底式引导，重新尝试

→ 4.4 动态支架：
    - 连续困难 → 降低布卢姆层级
    - 偏题 → 记录到 summary.md 的"⏳ 待解决"

→ 4.5 进度更新：
    - 更新 summary.md：核心结论、掌握状态
    - 更新 roadmap_status.md：进度、知识点状态、学习历史
    - 读取 Reflection 反馈 → 调整下一课难度

→ 循环回 4.2，直到用户退出
```

### Step 5: 生成学习报告

```
读取 summary.md + roadmap_status.md
→ 生成 report.md：目标达成度 + 掌握明细 + 能力图谱 + 进阶规划
→ ⛔ BLOCKING: 展示报告路径，等待反馈
```

## 九、文件状态机

描述三个核心文件在学习生命周期中的状态变迁：

```
┌───────────────────────────────────────────────────────┐
│                    学习会话生命周期                       │
├──────────┬───────────────┬───────────────┬────────────┤
│  阶段     │ summary.md    │ roadmap.md    │ lesson_N   │
├──────────┼───────────────┼───────────────┼────────────┤
│ 初始化    │ 从模板创建     │ 从模板创建     │ 不存在      │
│          │ 占位符文本     │ 全部 ⏳ 待学习  │            │
├──────────┼───────────────┼───────────────┼────────────┤
│ 生成L1   │ 不变          │ 不变          │ 创建，含讲   │
│          │              │              │ 解+空答案区  │
├──────────┼───────────────┼───────────────┼────────────┤
│ 用户答题  │ 不变          │ 不变          │ 用户填写     │
│          │              │              │ "我的答案"   │
├──────────┼───────────────┼───────────────┼────────────┤
│ 答案检查  │ 更新核心结论   │ 知识点→✅     │ 不变        │
│ 通过后   │ 更新掌握状态   │ 进度%更新     │            │
│          │ 更新学习路径   │ 里程碑打勾    │            │
│          │              │ 学习历史追加行  │            │
├──────────┼───────────────┼───────────────┼────────────┤
│ 生成L2   │ 不变          │ 不变          │ 创建新文件   │
├──────────┼───────────────┼───────────────┼────────────┤
│ ...循环   │ 逐步充实      │ 逐步推进      │ 逐个生成     │
├──────────┼───────────────┼───────────────┼────────────┤
│ 学习完成  │ 最终状态       │ 最终状态      │ 全部完成     │
│          │              │              │ + report.md │
└──────────┴───────────────┴───────────────┴────────────┘
```
