## 讲解内容

### 一、这个项目解决什么问题？

剪映有一个"智能剪口播"功能，但它有两个根本性的缺陷：

> **资料原文**：
> 剪映的"智能剪口播"有两个痛点：
> 1. **无法理解语义**：重复说的句子、说错后纠正的内容，它识别不出来
> 2. **字幕质量差**：专业术语（Claude Code、MCP、API）经常识别错误

[来源：`README.md` §为什么做这个？]

videocut-skills 的解法是：用 Claude 的**语义理解**解决第一个问题，用**自定义词典**解决第二个问题。

---

### 二、4 个 Skill 的分工

整个项目由 4 个 Skill 组成，各司其职：

| Skill | 职责 | 触发时机 |
|-------|------|---------|
| `安装` | 首次使用前的环境准备（Node.js、FFmpeg、API Key） | 首次使用 |
| `剪口播` | 核心：转录 + AI 口误分析 + 网页审核 + 剪辑 | 每次处理视频 |
| `字幕` | 字幕生成、校对、烧录 | 剪辑完成后 |
| `自进化` | 记录用户反馈，持续更新规则 | 发现问题时 |

[来源：`README.md` §Skill 清单]

---

### 三、整体数据流

这是一次完整处理的流程：

> **资料原文**：
> ```
> /videocut:安装  →  首次使用，安装环境和模型
>                     ↓
> /videocut:剪口播 视频.mp4
>   1. 提取音频 → 上传云端
>   2. 火山引擎转录 → 字级别时间戳
>   3. AI 审核：静音/口误/重复/语气词
>   4. 生成审核网页 → 浏览器打开
>                     ↓
> 【人工审核 + 执行剪辑】
>                     ↓
> /videocut:字幕
>   - 转录 → 词典纠错 → 人工确认 → 烧录字幕
> ```

[来源：`README.md` §使用流程]

关键点：整个流程是**人机协作**的，AI 负责识别和预选，人工负责最终确认。

---

### 四、核心数据格式：subtitles_words.json

整个剪口播流程的核心数据是 `subtitles_words.json`，它记录了每个字的精确时间戳：

> **资料原文**：
> ```json
> [
>   {"text": "大", "start": 0.12, "end": 0.2, "isGap": false},
>   {"text": "", "start": 6.78, "end": 7.48, "isGap": true}
> ]
> ```

[来源：`剪口播/SKILL.md` §数据格式]

每个元素有 4 个字段：
- `text`：字的内容（静音时为空字符串）
- `start` / `end`：开始和结束时间（秒）
- `isGap`：是否是静音段

这个格式是后续所有分析的基础——AI 分析口误、生成审核页面，都依赖它。

### 五、技术架构图

> **资料原文**：
> ```
> ┌──────────────────┐     ┌──────────────────┐
> │   火山引擎 ASR   │────▶│  字级别时间戳    │
> │  （云端转录）    │     │  subtitles.json  │
> └──────────────────┘     └────────┬─────────┘
>                                   │
>                                   ▼
> ┌──────────────────┐     ┌──────────────────┐
> │   Claude Code    │────▶│   AI 审核结果    │
> │  （语义分析）    │     │  auto_selected   │
> └──────────────────┘     └────────┬─────────┘
>                                   │
>                                   ▼
> ┌──────────────────┐     ┌──────────────────┐
> │   审核网页       │────▶│   最终删除列表   │
> │  （人工确认）    │     │  delete_segments │
> └──────────────────┘     └────────┬─────────┘
>                                   │
>                                   ▼
> ┌──────────────────┐     ┌──────────────────┐
> │     FFmpeg       │────▶│   剪辑后视频     │
> │  filter_complex  │     │   xxx_cut.mp4    │
> └──────────────────┘     └──────────────────┘
> ```

[来源：`README.md` §技术架构]

三个核心组件：
1. **火山引擎 ASR**：云端语音识别，生成字级别时间戳
2. **Claude Code**：语义分析，识别口误
3. **FFmpeg**：实际执行视频剪辑

## 本节要点总结

- videocut-skills 用语义理解（Claude）+ 自定义词典解决剪映的两大痛点
- 4 个 Skill：安装 → 剪口播 → 字幕 → 自进化，各司其职
- 核心数据格式 `subtitles_words.json`：每个字一条记录，含 text/start/end/isGap
- 整体是人机协作：AI 预选，人工最终确认
