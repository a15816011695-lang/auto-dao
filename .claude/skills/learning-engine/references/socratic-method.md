## 概述

苏格拉底教学法是一种以提问为核心的启发式教学方法，通过精心设计的问题引导学习者自主发现真理、构建知识。这种方法不直接给出答案，而是通过持续的追问，帮助学习者自己推导出结论。

**核心公式**：
```
提问 → 思考 → 回答 → 追问 → 深入思考 → 自我发现
```

---

## 核心理念

### 1. 产婆术 (Maieutics)

苏格拉底将自己比喻为"知识的产婆"——他不生产知识，而是帮助学习者"分娩"出内心已有的真理。

**教学启示**：
- 相信用户有能力自己得出答案
- 你的角色是引导，不是灌输
- 给予足够的时间和空间让用户思考

### 2. 无知之知

苏格拉底的名言："我唯一知道的就是我一无所知。"

**教学启示**：
- 以平等的姿态与用户对话
- 通过提问共同探索，而非自上而下的教导
- 承认知识的局限性，鼓励质疑

### 3. 否定式辩证法

通过指出矛盾和漏洞，帮助用户修正错误认知，逐步逼近真理。

**教学启示**：
- 不直接否定用户，而是引导用户发现自己的矛盾
- 将错误视为学习机会，而非失败

## 提问技巧

### 问题类型层次

#### Level 1: 事实性问题 (Factual Questions)
**目的**：确认基础信息

**示例**：
- "这个概念的定义是什么？"
- "文中提到了哪些关键术语？"

**使用时机**：学习初期，确认用户是否具备必要知识

---

#### Level 2: 解释性问题 (Explanatory Questions)
**目的**：促进理解

**示例**：
- "你能用自己的话解释这个概念吗？"
- "为什么这个公式是这个形式？"
- "这个过程是如何工作的？"

**使用时机**：确认用户理解概念的本质

---

#### Level 3: 推理性问题 (Inferential Questions)
**目的**：引导用户推导结论

**示例**：
- "如果这个前提成立，那么会得出什么结论？"
- "根据这些信息，你能推断出什么？"
- "这个结果意味着什么？"

**使用时机**：引导用户从已知推导未知

---

#### Level 4: 评估性问题 (Evaluative Questions)
**目的**：培养批判性思维

**示例**：
- "你认为这个观点有道理吗？为什么？"
- "这个方案的优缺点是什么？"
- "如果换一个条件，结论还成立吗？"

**使用时机**：帮助用户形成独立判断

---

#### Level 5: 综合性问题 (Synthetic Questions)
**目的**：整合知识，创造新见解

**示例**：
- "这两个概念有什么联系？"
- "你能将这个方法应用到其他领域吗？"
- "基于你学到的知识，你能提出一个新方案吗？"

**使用时机**：帮助用户融会贯通

## 教学对话模式

### 模式一：澄清式对话

**目的**：确保用户准确理解概念

**对话流程**：
```
AI: 你能告诉我什么是递归吗？
User: 就是一个函数调用自己。
AI: 很好。那么任何函数调用自己都是递归吗？
User: 嗯...好像不是，需要有终止条件？
AI: 为什么需要终止条件？
User: 否则会无限循环下去。
AI: 正确！所以递归的完整定义应该是什么？
User: 一个函数调用自己，并且有终止条件防止无限循环。
```

**技巧要点**：
- 从用户的回答出发继续追问
- 不直接纠正，而是引导用户自己发现问题

---

### 模式二：矛盾式对话

**目的**：帮助用户发现和修正认知矛盾

**对话流程**：
```
AI: 你说时间复杂度是 O(n)，能解释一下吗？
User: 因为循环执行了 n 次。
AI: 那循环内部的操作呢？
User: 是常数时间操作。
AI: 如果循环内部有一个排序操作呢？
User: 那就不是常数了...是 O(n log n)？
AI: 那整体复杂度是多少？
User: O(n² log n)... 我之前说得不对。
```

**技巧要点**：
- 不直接指出错误
- 通过追问让用户自己发现矛盾

---

### 模式三：类比式对话

**目的**：帮助用户理解抽象概念

**对话流程**：
```
AI: 你理解什么是"闭包"吗？
User: 不太理解，感觉很抽象。
AI: 想象你在家里放了一个存钱罐，然后你离开家去旅行。存钱罐还在那里，对吧？
User: 是的。
AI: 闭包就像那个存钱罐——函数离开创建它的环境后，仍然可以访问那个环境里的变量。
User: 所以闭包就是函数"记住"了它创建时的环境？
AI: 完美！你能想到一个使用场景吗？
```

**技巧要点**：
- 选择贴近用户生活的类比
- 类比后引导用户抽象出本质特征

## 注意事项

### 应该做的 ✓

1. **耐心等待**：给用户充分的思考时间，不要急于给出答案
2. **正向反馈**：对正确的方向给予肯定
3. **适时引导**：当用户卡住时，提供最小线索
4. **记录进度**：每次对话后更新学习状态

### 不应该做的 ✗

1. **不要直接给答案**：这违背苏格拉底教学法的核心
2. **不要连续追问太多问题**：一次只问一个关键问题
3. **不要忽视用户情绪**：如果用户困惑，及时降级
4. **不要偏离主线**：保持对话聚焦于学习目标

### 平衡原则

| 苏格拉底原则 | 实际应用 |
|-------------|---------|
| 不直接给答案 | 但可以给类比、提示 |
| 持续追问 | 但要关注用户承受能力 |
| 让用户自己发现 | 但要适时提供脚手架 |
