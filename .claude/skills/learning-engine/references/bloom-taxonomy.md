## 概述

布卢姆分类学是教育领域中用于分类教育目标的框架，将认知能力分为六个层级，从低阶到高阶依次递进。在学习引擎中，我们使用这一框架来设计渐进式学习路径。

```
      ┌─────────────────┐
      │   创造 (Create)   │  ← 最高阶
      ├─────────────────┤
      │   评价 (Evaluate) │
      ├─────────────────┤
      │   分析 (Analyze)  │
      ├─────────────────┤
      │   应用 (Apply)    │
      ├─────────────────┤
      │   理解 (Understand)│
      ├─────────────────┤
      │   记忆 (Remember) │  ← 最低阶
      └─────────────────┘
```

## 认知层级详解

### Level 1: 记忆 (Remember)
**定义**：从长期记忆中提取相关知识。

**行为动词**：
- 定义 (Define)
- 列举 (List)
- 识别 (Identify)
- 回忆 (Recall)
- 重复 (Repeat)

**教学提问示例**：
- "什么是...？"
- "你能列出...的主要特征吗？"
- "这个术语的定义是什么？"

**适合场景**：
- 学习新概念的初始阶段
- 确认用户是否具备基础知识

---

### Level 2: 理解 (Understand)
**定义**：从口头、书面和图像等交流形式的教学材料中建构意义。

**行为动词**：
- 解释 (Explain)
- 描述 (Describe)
- 概述 (Summarize)
- 分类 (Classify)
- 比较 (Compare)

**教学提问示例**：
- "你能用自己的话解释...吗？"
- "...和...之间有什么区别？"
- "为什么...会发生？"

**适合场景**：
- 确认用户理解概念的本质
- 建立概念间的联系

---

### Level 3: 应用 (Apply)
**定义**：在给定情境中执行或使用某个程序。

**行为动词**：
- 应用 (Apply)
- 实施 (Implement)
- 执行 (Execute)
- 解决 (Solve)
- 演示 (Demonstrate)

**教学提问示例**：
- "你能用...来解决这个问题吗？"
- "在什么情况下你会使用...？"
- "如果...，会发生什么？"

**适合场景**：
- 让用户在实践中巩固知识
- 检验用户能否迁移知识到新情境

---

### Level 4: 分析 (Analyze)
**定义**：将材料分解为组成部分，并确定各部分之间的关系及其与整体结构的关系。

**行为动词**：
- 分析 (Analyze)
- 区分 (Differentiate)
- 组织 (Organize)
- 比较 (Compare)
- 归因 (Attribute)

**教学提问示例**：
- "...的组成部分是什么？"
- "你能找出这个论证中的逻辑漏洞吗？"
- "这与其他方法相比有什么优势和劣势？"

**适合场景**：
- 深入理解复杂系统
- 培养批判性思维

---

### Level 5: 评价 (Evaluate)
**定义**：基于标准和准则做出判断。

**行为动词**：
- 评价 (Evaluate)
- 判断 (Judge)
- 批评 (Criticize)
- 辩护 (Defend)
- 论证 (Argue)

**教学提问示例**：
- "你认为这个方案合理吗？为什么？"
- "哪个选项更好？你的标准是什么？"
- "你能找出这个论证的问题吗？"

**适合场景**：
- 培养用户独立判断能力
- 让用户形成自己的观点

---

### Level 6: 创造 (Create)
**定义**：将要素组装在一起，形成一个连贯或功能的整体；将要素重组为新的模式或结构。

**行为动词**：
- 创造 (Create)
- 设计 (Design)
- 开发 (Develop)
- 构建 (Construct)
- 提出 (Propose)

**教学提问示例**：
- "你能设计一个...吗？"
- "如果让你从头构建...，你会怎么做？"
- "你能提出一个改进方案吗？"

**适合场景**：
- 最高阶的知识内化
- 让用户产出原创性成果

## 教学应用指南

### 渐进式提问策略

学习引擎应按照布卢姆层级逐步提升问题难度：

```
Lesson 开始
    │
    ▼
┌─────────────┐
│  记忆/理解   │ ← 确认基础
└─────────────┘
    │ 用户回答正确
    ▼
┌─────────────┐
│   应用      │ ← 实践检验
└─────────────┘
    │ 用户回答正确
    ▼
┌─────────────┐
│  分析/评价   │ ← 深入探索
└─────────────┘
    │ 用户回答正确
    ▼
┌─────────────┐
│   创造      │ ← 综合产出
└─────────────┘
```

### 问题设计原则

1. **单一焦点**：每个问题聚焦一个认知层级
2. **清晰边界**：问题不要跨越多个层级
3. **循序渐进**：确保用户具备低层级能力后再推进

## 动态降级策略

### 触发条件

当用户出现以下情况时，自动降低认知层级：
- 连续两次回答"不知道"或"不清楚"
- 回答明显偏离正确方向
- 表现出困惑或沮丧情绪

### 降级路径

```
分析/评价/创造 → 应用 → 理解 → 记忆
```

### 降级后的教学策略

| 原层级 | 降级后策略 |
|--------|-----------|
| 创造/评价 | 提供类比或案例让用户理解 |
| 分析 | 提供分步引导，降低任务复杂度 |
| 应用 | 回到概念解释，使用类比说明 |
| 理解 | 提供直接的定义和示例 |

### 示例

**原问题（分析层级）**：
"你能分析这个算法的时间复杂度为什么是 O(n²) 吗？"

**用户困惑后降级（理解层级）**：
"让我换个方式。你记得什么是嵌套循环吗？外层循环执行 n 次，内层循环也执行 n 次，总共执行多少次？"
