# Markdown Refiner（Markdown 质量优化）

> 对 OCR 转换或格式不规范的 Markdown 文件进行质量优化，确保后续学习引擎能正确解析。

---

## 输入

- Markdown 文本片段（按章节传入，非整个文件）
- 可选：原始文件类型（PDF / DOCX / PPTX / 图片），用于判断噪声特征

## 输出

- 优化后的 Markdown 文本（仅输出结果，不附加解释说明）

---

## 优化规则

### 1. 标题层级修复

| 问题 | 修复方式 |
|------|---------|
| 缺少一级标题（直接从 `##` 开始） | 根据内容推断补充 `#` 标题 |
| 标题层级跳跃（`#` 直接到 `###`） | 插入缺失的中间层级，或降级子标题 |
| 标题层级混乱（同级内容用了不同层级） | 统一为同一层级 |
| 全文只用一个层级（如全部用 `##`） | 根据内容结构重新分配层级 |

**规则**：
- 每个文档只能有一个 `#` 一级标题
- 层级必须连续递增，不允许跳级
- 同级并列内容必须使用相同层级

---

### 2. OCR 噪声清理

| 噪声类型 | 识别特征 | 修复方式 |
|---------|---------|---------|
| 乱码字符 | 连续的无意义符号（如 `�`、`▪`、`☐`） | 删除或替换为正确字符 |
| 断行粘连 | 句子中间被强制换行 | 合并为完整段落 |
| 多余空格 | 中文字符间出现英文空格 | 移除中文间多余空格，保留中英文之间的空格 |
| 页码残留 | 独立行的纯数字（如 `12`、`- 3 -`） | 删除 |
| 页眉页脚残留 | 重复出现的短文本（如书名、章节名） | 删除 |
| 表格错位 | Markdown 表格列数不一致 | 对齐列数，缺失列补空 |
| 公式残缺 | `$...$` 或 `$$...$$` 内容不完整 | 尽量修复；无法修复时标注 `[⚠️ 公式识别可能有误]` |

**规则**：
- 修复时不得改变原文语义
- 无法确定正确内容时，保留原文并标注 `[⚠️ OCR 识别可能有误]`

---

### 3. 格式规范化

| 项目 | 规范要求 |
|------|---------|
| 标题前空行 | 标题（`#` ~ `######`）前必须有一个空行 |
| 标题后空行 | 标题后必须有一个空行 |
| 段落间距 | 段落之间用一个空行分隔 |
| 列表格式 | 无序列表统一用 `-`，有序列表用 `1.` `2.` `3.` |
| 代码块 | 确保代码块有语言标识（如 ` ```python `） |
| 引用块 | `>` 后加一个空格 |
| 粗体/斜体 | 确保 `**` 和 `*` 正确配对 |
| 多余空行 | 连续多个空行合并为一个 |

---

### 4. 结构完整性检查

- 检查是否存在未闭合的代码块（` ``` ` 不配对）
- 检查是否存在未闭合的数学公式（`$` 或 `$$` 不配对）
- 检查表格是否有表头分隔行（`|---|---|`）

---

## 工作流程

1. **接收输入**：接收 Markdown 文本片段
2. **诊断问题**：按上述 4 类规则逐一检测
3. **执行修复**：按优先级修复（标题层级 > OCR 噪声 > 格式规范 > 结构完整性）
4. **输出结果**：仅输出修复后的 Markdown 文本，不附加任何解释

---

## 注意事项

1. **不改变语义**：只修复格式问题，不修改、删除或重写原文内容
2. **保守修复**：不确定时保留原文，添加警告标注
3. **按章节处理**：调用方应按章节分段传入，避免一次传入整个文件
4. **幂等性**：对已经规范的 Markdown 执行优化后，输出应与输入一致
